# UAT / Staging Bug Regressions — 2026-06

Four confirmed bugs, one root cause: the system does not call `EvaluateRemainingPlans()` after NTU/Decline/Postpone. Every bug below **must** have a regression test that stays green forever.

## Common root fix
Any handler that mutates a plan's status must fire the `RiderStatusChanged` domain event → subscribed `RiderStatusChangedHandler` runs `IRemainingPlansEvaluator` → applies `EvaluationResult` to the policy → generates letters and reminders. If a handler skips this path, the architecture test from PH-06-05 fails the build.

## BUG-2610000310P — Substatus stuck after NTU Exclusion rider

- **Environment:** UAT.
- **Setup (source lines 465-468, 1974-1981):**
  - Base Plan = Standard
  - Premier Rider = Exclusion → NB CLOA (Exclusion) generated
  - Added Choice Rider + Cancer Guard Rider
  - Choice Rider = Decline (Standard)
  - Cancer Guard = Standard
- **Action:** Perform NTU for Premier Rider (Exclusion).
- **Current bug:** Substatus stuck at `ConditionalAcceptanceLetterGenerated`. NB CLOA (Exclusion) Reminder keeps generating with Premier Rider (NTU'd) still in the Exclusion table. User can still click Accept CLOA = Yes.
- **Expected:**
  - Substatus → `PendingCashCollection`
  - New NB LOA generated for Base Plan + Cancer Guard Rider
  - CLOA Reminder STOPS (remaining plans are all Standard)
  - Accept CLOA field greyed out
- **Root cause:** `EvaluateRemainingPlans()` not called after NTU Premier Rider.
- **Root fix:** NTU Premier → `RiderStatusChanged` event → `RemainingPlansEvaluator` → Scenario A (AllStandard) → substatus = PendingCashCollection (shortfall > 0) → RCMP fields greyed → new LOA → cancel CLOA reminders.
- **Regression test ID:** `Bug_2610000310P_NtuPremierExclusion_YieldsPendingCashAndNewLoa`.
- **Test location:** `MOHProject.Tests/Regression/UatBugs/Bug_2610000310P_Test.cs`.

## BUG-2610000154P — RCMP fields not auto-reset after NTU RCMP rider

- **Environment:** Staging.
- **Setup (source lines 471-473, 1987-1994):**
  - Base Plan = Standard
  - Choice Rider = RCMP
  - NB CLOA (RCMP) generated
  - UW selected Option 1 + Accepted CLOA
  - Substatus = `PendingCashCollection`
- **Action:** Perform NTU for Choice Rider (RCMP).
- **Current bug:** Substatus stuck at `PendingCashCollection`. RCMP Flag still ticked. Accept CLOA still Yes. RCMP Option still Option 1. Private Insurance Extra Premium for Choice Rider auto-zeroed (wrong). Gross Premium for Choice Rider excludes Risk Loading (wrong).
- **Expected:**
  - Substatus → `PendingIpRequestFile`
  - RCMP Flag untick
  - Accept CLOA = Blank
  - RCMP Option = Blank
  - Private Insurance Extra Premium and Gross Premium of Choice Rider **RETAIN** their pre-NTU values (used for historical display — see FR-AOR-030 note)
- **Root cause:** Same as above.
- **Root fix:** `RemainingPlansEvaluator` → Scenario A (AllStandard because only Base=Standard remains) → grey the 3 RCMP fields → substatus = `PendingIpRequestFile` (no shortfall, SG/PR × SG/PR).
- **Regression test ID:** `Bug_2610000154P_NtuChoiceRcmp_ResetsRcmpFieldsAndRetainsHistoricalPremiums`.
